
【增加行情订阅流程】
1. 在函数loginWind()中增加订阅；
2. RealTimeTick类中增加储存该品种实时行情字段；
3. 修改回调函数，存储该品种行情；
4. 让行情显示在tableView中。



【增加策略流程】
1. 先增加策略相关品种行情订阅；
2. 参考portf_Arbitrage设计策略类及策略UI；
3. 在MainWindow类中增加该策略指针，并在构造函数中布局显示。

让回调函数只负责行情存储及策略参数计算，开一个新线程负责数据库数据的显示，二者必须独立。


【程序结构设计】
1. 两个全局类对象：configInfo.h、RealTimeTick.h，一个存储配置文件信息，程序运行中不会修改；另一个存储订阅的实时行情，程序运行中实时更新。

2. 对股票、期货、和指数分别定义回调函数，因为订阅的字段各不相同（这样做的必要性待讨论）。

3. 说一下MainWindow.h这个类：这个类是程序的主界面，界面左侧是用来加载策略的（portf_xx)；界面右侧分为上下两个部分，上面是一个QTableView，用来显示最新的行情信息，通过最新的行情时间可判断程序是否运行正常，下面是一个QTextEdit，程序运行中出现的一些提示性信息可在这里显示。

4. 策略类：portf_xx.h，以portf_开始命名的都是策略类，是QWidget类的子类，每一个策略的ui和算法都在这个类里面实现。该类中portf_set成员负责存储该策略相关品种，update()函数负责策略的算法并将需要显示的东西通过UI实时更新。

5. 程序启动，创建MainWindow，登录wind并订阅行情，行情通过回调函数返回并更新RealTimeTick，同时调用MainWindow的update()函数，该函数通过判断品种属于哪一个策略，并调用对应策略的update()函数。
